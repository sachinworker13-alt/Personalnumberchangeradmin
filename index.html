🛡 Security Features Status
🔐
SHA-256 Password Hashing
All credentials are hashed client-side using SHA-256 before being stored in Firebase. Plaintext passwords are never stored.
ACTIVE
🕐
Session Expiry (4 hours)
User sessions automatically expire after 4 hours. Sessions are signed with a cryptographic token to prevent tampering.
ACTIVE
🚫
Brute Force Lockout
Accounts are locked for 30 minutes after 5 consecutive failed login attempts. Failed attempts are logged to Firebase.
ACTIVE
🖥️
Device License Approval
New devices must be approved by the admin before they can log in. Devices auto-register on first login attempt and remain pending until approved.
ACTIVE
🔍
DevTools Detection
The tool detects open browser developer tools and blurs/blocks the interface to prevent inspection and code extraction.
ACTIVE
📋
Right-Click & Copy Protection
Context menus, text selection, Ctrl+U (view source), Ctrl+S (save), and drag operations are all disabled.
ACTIVE
🔥
Firebase Realtime Sync
Credentials and session data sync in real-time via Firebase. Credential changes and force-logout propagate instantly to all sessions.
CHECKING…
🔑
Debugger Trap
A debugger trap runs every 3 seconds. If developer tools are actively used to step through code, the tool interface is wiped.
ACTIVE
